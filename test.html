Object-Oriented JavaScript (OOJS) is a programming paradigm that uses objects to structure code. While JavaScript is not a "classical" object-oriented language like Java or C++, it still supports OOP concepts like inheritance, encapsulation, and polymorphism, primarily through objects and prototypes.

Key Concepts in Object-Oriented JavaScript

  1. Objects: In JavaScript, almost everything is an object (functions, arrays, etc.). Objects can have properties (data) and methods (functions). 

    javascript
    const person = {
  name: 'John',
  age: 30,
  greet: function() {
    console.log('Hello, ' + this.name);
  }
};
person.greet(); // Output: Hello, John

  2. Prototypes: JavaScript is prototype-based, meaning every object has a prototype object from which it can inherit properties and methods. Prototypes allow shared functionality across multiple objects. 

    javascript
    function Person(name) {
  this.name = name;
}
Person.prototype.greet = function() {
  console.log('Hello, ' + this.name);
};
const john = new Person('John');
john.greet(); // Output: Hello, John

  3. Classes (ES6+): With ES6, JavaScript introduced the class syntax to make OOP more accessible and familiar to developers coming from other OOP languages. However, classes in JavaScript are syntactical sugar over prototypes. 

    javascript
    class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }
  greet() {
    console.log(`Hello, my name is ${this.name}`);
  }
}
const jane = new Person('Jane', 25);
jane.greet(); // Output: Hello, my name is Jane

  4. Inheritance: In JavaScript, one class (or function constructor) can inherit properties and methods from another, allowing for reusable code. 

    javascript
    class Employee extends Person {
  constructor(name, age, jobTitle) {
    super(name, age); // Calls parent class constructor
    this.jobTitle = jobTitle;
  }
  work() {
    console.log(`${this.name} is working as a ${this.jobTitle}`);
  }
}
const bob = new Employee('Bob', 28, 'Software Developer');
bob.greet(); // Output: Hello, my name is Bob
bob.work();  // Output: Bob is working as a Software Developer

  5. Encapsulation: JavaScript doesn't have private properties by default, but you can use closures or symbols to simulate them. With ES2022 (ES12), private class fields are officially supported using #. 

    javascript
    class Person {
  #ssn; // private field
  constructor(name, ssn) {
    this.name = name;
    this.#ssn = ssn;
  }
  getSSN() {
    return `SSN is private for ${this.name}`;
  }
}
const person1 = new Person('Alice', '123-45-6789');
console.log(person1.getSSN()); // Output: SSN is private for Alice

3 Challenges to Solve

1. Create a Simple Class with Methods and Properties

Write a class Car with the following features:

  • Properties: brand, model, year.
  • Methods:
    • start() that prints "The [brand] [model] is starting".
    • stop() that prints "The [brand] [model] is stopping".

Bonus: Add a method age() that calculates how old the car is based on the current year.

2. Implement Inheritance

Create a base class Animal with properties name and a method speak() that prints "[name] makes a sound". Then, create a derived class Dog that inherits from Animal, and override the speak() method to print "[name] barks".

Bonus: Add another method fetch() in the Dog class that prints "[name] is fetching the ball".

3. Encapsulation with Private Fields

Create a class BankAccount that encapsulates:

  • Private fields: balance.
  • A method deposit(amount) to add money to the balance.
  • A method withdraw(amount) to subtract from the balance, ensuring that it doesn’t go negative.
  • A method getBalance() to return the current balance but make sure the balance can’t be accessed directly.
